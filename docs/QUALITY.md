# Quality Standards — tangyuan-farming

## Sprint 评估标准（Anthropic 启发式）

> 灵感来自 Anthropic Labs 的长运行 Agent harness 设计
> 每个 Sprint 结束后必须通过以下 4 个维度评估，**任何一项低于及格线 → 打回重做**

| 维度 | 权重 | 及格线 | 评估方法 | 负责人 |
|------|------|--------|---------|--------|
| **功能完整性** | 40% | 80 分 | 所有需求的功能点都实现 | Generator |
| **代码质量** | 30% | 70 分 | 无 lint 错误、类型正确、无 bug | Evaluator |
| **视觉设计** | 20% | 60 分 | 符合设计约定，无 AI 生成痕迹 | Evaluator |
| **测试覆盖** | 10% | 80% | 核心路径有测试覆盖 | Generator |

### 评分细则

#### 功能完整性（40 分）
- ✅ **90-100 分**：所有功能点实现，超出预期
- ✅ **80-89 分**：所有功能点实现，符合预期
- ❌ **70-79 分**：核心功能实现，次要功能缺失
- ❌ **<70 分**：核心功能缺失 → **打回**

#### 代码质量（30 分）
- ✅ **90-100 分**：代码优雅、注释完整、无警告
- ✅ **80-89 分**：代码清晰、无 lint 错误
- ✅ **70-79 分**：代码可运行，有小问题
- ❌ **<70 分**：有 bug 或 lint 错误 → **打回**

#### 视觉设计（20 分）
- ✅ **90-100 分**：设计精美、有创意
- ✅ **80-89 分**：设计整洁、符合规范
- ✅ **70-79 分**：设计普通、无大错
- ❌ **<70 分**：设计粗糙 → **打回**

#### 测试覆盖（10 分）
- ✅ **90-100%**：核心路径 + 错误路径全覆盖
- ✅ **80-89%**：核心路径覆盖
- ❌ **<80%**：测试不足 → **打回**

## What to Test

### 后端（Express + Supabase）

- ✅ **成功路径** — API 正常返回 200
- ✅ **错误路径** — 401 未授权、404 不存在、500 服务器错误
- ✅ **边界条件** — 空列表、超大数值、特殊字符
- ✅ **安全验证** — JWT 过期、验证码失效

### 前端（Vue 3）

- ✅ **组件渲染** — 页面正常加载
- ✅ **用户交互** — 按钮点击、表单提交
- ✅ **API 调用** — 成功/失败处理

## Security-Sensitive Functions

### 后端（必须验证）

| 函数 | 验证要求 | 位置 |
|------|---------|------|
| `sendCode()` | 限制发送频率（60秒间隔） | `api/send-code.js` |
| `verifyCode()` | 验证码一次有效、5分钟有效 | `api/login.js` |
| `allocateLand()` | 验证用户已认证 | `api/allocate-land.js` |

### 前端（必须验证）

| 操作 | 验证要求 | 位置 |
|------|---------|------|
| 表单提交 | 前端验证 + 后端验证 | 所有 `*View.vue` |
| 路由跳转 | 登录状态检查 | `router/index.ts` |

## Code Review Checklist

### 后端

- [ ] 所有 API 都有身份验证或明确标注为公开接口
- [ ] 验证码有时效性检查（5分钟）
- [ ] 错误返回统一格式 `{ code, message, data }`
- [ ] 敏感操作有日志记录

### 前端

- [ ] 所有 API 调用有错误处理（try-catch）
- [ ] 表单有前端验证（必填项、格式检查）
- [ ] 路由有权限守卫（需要登录的页面）
